# 三层分类总结系统设计文档

> 版本: v2.0 | 创建时间: 2026-01-02 | 状态: 设计中

## 📌 项目背景

当前的总结系统使用通用 Prompt 处理所有类型的视频/播客内容，导致总结质量参差不齐。本方案通过三层分类框架，实现**内容感知的智能总结**。

---

## 🎯 设计目标

1. 让 AI 自动识别内容的三个维度
2. 根据识别结果动态组合最优总结策略
3. 保留旧逻辑，支持一键切换

---

## 📊 三层分类框架

### 第一层：内容形式 (Content Form)

> "这是什么类型的内容？"

| ID | 类型 | 典型特征 |
|----|------|---------|
| `tutorial` | 教程课程 | 有明确教学意图，知识传授 |
| `interview` | 访谈对话 | 多人互动，问答形式 |
| `monologue` | 独白演讲 | 单人表达观点/经历 |
| `news` | 新闻资讯 | 事件报道，时效性强 |
| `review` | 评测分析 | 对产品/事物的评价 |
| `finance` | 财经投资 | 市场分析、投资观点 |
| `narrative` | 故事叙事 | Vlog、经历、纪录 |
| `casual` | 闲聊杂谈 | 无明确主题，随意聊天 |

### 第二层：信息结构 (Information Structure)

> "内容是如何组织的？"

| ID | 结构类型 | 描述 |
|----|---------|------|
| `hierarchical` | 层级结构 | 概念→子概念→细节 |
| `sequential` | 顺序流程 | 步骤1→步骤2→步骤3 |
| `argumentative` | 论证链条 | 论点→论据→结论 |
| `comparative` | 对比分析 | A vs B 多维度对比 |
| `narrative_arc` | 叙事弧线 | 起因→发展→高潮→结局 |
| `thematic` | 主题罗列 | 话题A、话题B 平行展开 |
| `qa_format` | 问答格式 | 问题→回答交替 |
| `data_driven` | 数据驱动 | 基于数据/事实分析 |

### 第三层：认知目标 (Cognitive Goal)

> "用户想从中获得什么？"

| ID | 认知目标 | 总结导向 |
|----|---------|---------|
| `understand` | 理解学习 | 提取可迁移知识 |
| `decide` | 决策支持 | 提取结论+论据+风险 |
| `execute` | 行动指南 | 提取可执行步骤 |
| `inspire` | 观点启发 | 提取独特见解/金句 |
| `digest` | 快速摘要 | 提取事实精华 |

---

## ⚡ 策略切换机制

通过环境变量一键切换新旧策略：

```bash
# backend/.env
SUMMARY_STRATEGY=legacy         # 使用旧的通用 Prompt（默认）
SUMMARY_STRATEGY=v2_classified  # 使用新的三层分类系统
```

---

## 🔧 技术实现要点

### 分类器输出格式

```json
{
  "content_form": "interview",
  "info_structure": "argumentative",
  "cognitive_goal": "decide",
  "confidence": 0.85
}
```

### Prompt 模块化组合

```
[通用指令] + [信息结构模版] + [认知目标模版] + [内容形式补充]
```

### 关键文件

- `backend/prompts.py` - 分类器和模版定义
- `backend/summarizer.py` - 分类和动态组合逻辑
- `backend/config.py` - SUMMARY_STRATEGY 配置

## 📈 效果量化

### 评估维度（1-5 分）

| 维度 | 说明 | 权重 |
|------|------|------|
| 完整性 | 是否覆盖主要信息 | 25% |
| 准确性 | 是否如实反映原文 | 25% |
| 结构性 | 组织是否清晰连贯 | 20% |
| 价值密度 | 信息量是否足够 | 20% |
| 可读性 | 是否便于快速浏览 | 10% |
