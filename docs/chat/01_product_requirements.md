# 产品需求文档 (PRD): 聊天会话 (Thread) 管理 (单一会话模式)

## 1. 目标 (Goal)
建立一个完整的“视频对话”功能。**支持多会话并存**。无需“归档”状态，所有非删除会话均可见。用户可以在侧边栏自由切换不同话题。

## 2. 问题陈述 (Problem Statement)
- **现状**: 代码库目前缺乏有效的聊天模块后端支持，但前端已有 Sidebar 组件雏形。
- **目标**: 实现完整的会话管理后端，并连接前端 Sidebar，提供标准的聊天体验。

## 3. 用户故事 (User Stories)
1.  **继续对话**:
    -   作为用户，我可以从侧边栏选择一个历史会话，查看记录并继续对话。
2.  **新对话 (New Chat)**:
    -   作为用户，我可以随时点击“新对话”开启一个新话题，而不丢失旧话题。
3.  **管理会话**:
    -   作为用户，我可以在侧边栏看到所有未删除的会话列表。

## 4. 功能需求 (Functional Requirements)

### 4.1. 会话创建与显示 (Thread Creation & Display)
-   **进入页面**:
    -   默认加载最近一次活跃的会话，或显示“空状态”引导创建新会话。
-   **多会话支持**:
    -   允许同一用户针对同一视频创建多个会话。
-   **新建**:
    -   点击 "New Chat" 清空当前视图，发送消息时创建新 Thread 记录。

### 4.2. 会话存储与归档 (Thread Storage & Archiving)
### 4.2. 会话存储与删除 (Thread Storage & Deletion)
-   **状态**: `active` (默认), `deleted` (软删除)。**不使用 `archived` 状态**。
-   **列表逻辑**: API 默认返回所有 `status != 'deleted'` 的会话，按 `updated_at desc` 排序。
-   **删除**: 用户手动删除会话时，执行软删除 (`status='deleted'`)。

### 4.3. 前端界面 (Frontend Interface)
-   **侧边栏 (Sidebar)**:
    -   显示历史会话列表 (按更新时间排序)。
    -   支持点击切换。
-   **Header**:
    -   显示当前状态/标题。
    -   New Chat 按钮。

## 5. 非功能性需求 (Non-Functional Requirements)
-   **简洁性**: 交互逻辑必须呈现为线性的（Current or New），降低用户认知负担。
-   **扩展性**: 数据库设计需兼容未来可能加入的“历史记录列表”功能。
